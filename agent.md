# 功能设计文档

## 功能概述

本项目包含两个主要功能：
1. **账单管理功能**：
   - 目标：记录用户的账单信息。
   - 输入：用户通过自然语言输入账单信息，例如“请帮我在账单上加今天去吃饭，花了50块钱”。
   - 输出：无直接输出，仅保存数据。

2. **每日追踪功能**：
   - 目标：记录用户的每日活动。
   - 输入：用户通过自然语言输入活动信息，例如“记录今天完成了跑步5公里”。
   - 输出：无直接输出，仅保存数据。

---

## 输入格式化

### 1. 账单管理功能
- **输入示例**：
  ```
  请帮我在账单上加今天去吃饭，花了50块钱
  ```
- **解析逻辑**：
  - 使用自然语言处理（NLP）工具或正则表达式解析用户输入。
  - 提取关键信息：日期、类别、描述、金额。
- **格式化后数据结构**：
  ```json
  {
    "date": "2026-02-19",
    "category": "餐饮",
    "description": "去吃饭",
    "amount": 50
  }
  ```

### 2. 每日追踪功能
- **输入示例**：
  ```
  记录今天完成了跑步5公里
  ```
- **解析逻辑**：
  - 使用自然语言处理（NLP）工具或正则表达式解析用户输入。
  - 提取关键信息：日期、活动类型、详细信息。
- **格式化后数据结构**：
  ```json
  {
    "date": "2026-02-19",
    "activity": "跑步",
    "details": "5公里"
  }
  ```

---

## 数据存储

- **存储方式**：
  - 数据可以存储为JSON文件，或者使用SQLite数据库。
  - 例如：
    - 账单管理数据存储在 `bills.json` 文件中。
    - 每日追踪数据存储在 `daily_tracker.json` 文件中。

---

## Agent 实现

- **输入处理**：
  - 设计一个统一的接口，接收用户的自然语言输入。
  - 根据输入内容判断功能类型（账单管理或每日追踪）。
  - 调用对应的解析逻辑。

- **数据存储**：
  - 将解析后的数据存储到对应的文件或数据库中。

- **扩展性**：
  - 未来可以为这些功能添加输出逻辑，例如生成报表或图表。
